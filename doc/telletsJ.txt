telletsJ 即 tellets Java 的移植

数据源, 元信息 和 原来 tellets 完全一样

不一样的地方

telletsJ
	不需要必须持久化在磁盘, 因为很多环境下是不允许访问本地磁盘的
	即在内存中做文章的缓存
	考虑使用 rest 接口,并且允许CROS, 这样我不需要把页面部署在服务器上, 例如我的空间只支持 php
	可以运行直接生成本地静态数据文件
		例如: java -jar telletsj.jar export xxxx
	

定位:
	博客呈现
	只作为一个前端,不操作数据,不编辑数据,只呈现数据
	支持多种文章格式
	预编译为静态文件或不就行预编译
	缓存文件
	不操作实际文件, 这里需要做一个权衡
		当然也可以实现一个数据端
	但是写前端也并不是我的什么强项
	更考虑实现一个简单的 rest server
	这样前端就好写多了~
	
	
基本目标:
	能替换掉现有的 tellets
	
	

Telletsj
管理模块
	manage
  管理
  配置
收集模块
	collect
	最主要的目的就是收集内容
	common-vfs 在这里很适合
	最主要的收集目标
		GitHub
		在common-vfs的基础上实现GitHub的访问,包括授权

处理模块
	process
	获取文章基本信息
		连接
插件扩展模块
	plugin
实体对象模块
	article
核心模块
	core
	关联以上所有的模块,其实成为一个整体
